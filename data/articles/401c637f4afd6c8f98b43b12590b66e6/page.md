# 【早阅】别被反AI的热潮带偏了：Antirez的AI编程实践与思考

前言

开源大神 antirez 反思人工智能时代的编程变革 —— 代码的未来不在手指，而在思维；拥抱 AI，才能在新时代继续创造。今日前端早读课文章由 @antirez 分享，@飘飘编译。

译文从这开始～～

我喜欢一行一行地编写代码。可以说，我整个职业生涯一直都在追求写出优雅、简洁、带有人文气息的软件。我也希望生活在一个不忘弱者的社会中。更坦白地说，我并不希望 AI 在经济上取得成功，我也不在意现行的经济体系是否会被颠覆 —— 如果能促成财富的大规模再分配，我甚至会感到高兴。但是，如果我对软件和社会的看法会妨碍我的视野，那我就是不尊重自己和自己的智慧：事实就是事实：AI 正在永远地改变编程。

在 2020 年，我辞去了工作，开始写一本关于人工智能、全民基本收入，以及社会如何适应自动化所带来挑战的小说。到 2024 年底，我开设了一个 YouTube 频道，专注于 AI 在编程中的应用，以及它可能带来的社会与经济影响。虽然我很早就意识到这种变化的趋势，但我原以为我们还有几年时间，编程才会被彻底重塑。如今我不再这么想。最新的顶尖大语言模型（LLM）已经能在极少辅助的情况下，独立完成大型子任务或中等规模项目 —— 只要你能清楚描述想要的结果。成功的程度取决于你所做的编程类型（越独立、越能用文本表达的部分效果越好，系统编程尤其适合），以及你能否清晰地构建问题的思维模型并传达给模型。但总体来说，如今对大多数项目而言，亲自写代码已经没有太大意义了，除非是为了好玩。

在过去的一周里，仅靠提示和偶尔检查代码提供指导，我在几个小时内完成了过去要花上几周的四件事：

1、我修改了自己的 linenoise 库，让它支持 UTF-8，还创建了一个基于模拟终端的行编辑测试框架，这个终端可以报告每个字符格子里显示的内容。过去我一直想做这件事，但很难为这样一个小项目投入那么多时间。而如今，只要描述想法，它就能自动变成代码，情况完全不一样。

2、我修复了 Redis 测试中的偶发性错误。这类工作非常恼人，涉及时间同步、TCP 死锁等问题。Claude Code 一直在反复测试，分析进程状态，直到找到问题并修复。

3、昨天我想要一个纯 C 的库，用来推理类似 BERT 的嵌入模型。Claude Code 用 5 分钟就写好了。性能与 PyTorch 相当，只慢约 15%。代码 700 行，还附带一个 Python 工具来转换 GTE-small 模型。

4、最近几周我在修改 Redis Streams 的内部结构。我原本写了一个设计文档，然后交给 Claude Code，它花了不到 20 分钟就完整复现了我的工作（主要时间花在我确认和批准执行命令上）。

根本无法忽视正在发生的事实。在大多数情况下，不再需要编写代码。更有价值的，是思考 “要做什么” 和 “怎么做”。而在 “怎么做” 这方面，LLM 也是绝佳的合作伙伴。无论 AI 公司能否收回投资、股市是否崩盘，这些都无关紧要。从长远来看，这场变革已经不可逆转。无论某些独角兽 CEO 说出多么荒谬或刺耳的话，编程已被彻底改变。

那我对自己的代码被 LLM 用来训练这件事怎么看？我觉得很好。因为这正是我毕生追求的延续 —— 让代码、系统与知识更加民主化。LLM 将帮助我们写出更好的软件，更快、更高效，也让小团队有机会与大公司竞争。这与上世纪 90 年代开源软件的意义如出一辙。

不过，这项技术实在太重要了，不该只掌握在少数公司手中。目前，虽然有人在预训练和强化学习方面做得更好，但开源模型 —— 尤其是来自中国的那些 —— 依然能与封闭实验室的前沿模型竞争（即使稍有落后）。目前的 AI 仍处于相对 “民主化” 的阶段，尽管并不完美。但没人能保证这种状况会一直持续。我担心集中化的趋势。与此同时，我也坚信大规模神经网络的潜力是惊人的，而且当前的前沿模型并没有什么 “神秘魔法” 不可复制 —— 否则也很难解释为什么这些年来 OpenAI、Anthropic 和 Google 的成果一直相差无几。

作为程序员，我现在比以往更想写开源代码。我想重新打理那些因为时间不足而荒废的仓库，把 AI 引入我的 Redis 开发流程，改进 Vector Sets 的实现，然后优化更多数据结构，比如我正在改的 Streams。

但我也为那些即将失业的人感到担忧。未来会怎样，还不确定：公司会雇更多人去创造更多东西？还是会为了节省工资，只留下少数擅长提示工程的人？我担心，在许多行业中，人类将被彻底取代。

那么，社会的出路是什么？创新无法被阻止。我们应该投票支持那些理解现状、愿意帮助失业者的政府。而且，随着失业人数增加，政治上也会出现更多支持保障措施的呼声。与此同时，我仍然期待 AI 带来的好处：它可能推动科学进步，帮助人类减轻痛苦，让生活不再那么艰难。

不管怎样，还是回到编程上来吧。我给你一个建议，我的朋友。无论你认为正确的事情应该是什么样，你都无法通过拒绝当下正在发生的事情来控制它。回避人工智能对你和你的职业生涯都没有帮助。好好想想吧。谨慎地测试这些新工具，花上几周的时间，而不是五分钟就草草了事，这样只会让你更加坚信自己的看法。想办法让自己变得更强，如果这对你不起作用，每隔几个月就再试一次。

也许你会想：“我辛苦学了编程，如今机器却替我写代码。” 但回想一下，当年你熬夜写代码、看到项目运行成功时，那种热情是什么？是 “创造”。而现在，如果你学会善用 AI，你能创造得更快、更好。那份乐趣，其实从未消失。
